# リスクマネージャー 状態遷移図

[English](risk-state-machine.md) | **日本語**

## 概要

リスク管理システムは、各トレーディングサイクルでボットが何をすべきかを決定します。複数の条件が同時に発火した場合、最も深刻なアクションが常に優先されます。

## リスクレベル (RISK_LEVEL 環境変数)

ボットを再起動せずにランタイムで変更可能。

| レベル   | サイズ倍率 | 動作                                |
|----------|----------:|--------------------------------------|
| `green`  | 1.0x      | 通常取引                             |
| `yellow` | 0.5x      | ポジションサイズ半減                 |
| `red`    | 0.0x      | 一時停止 — 新規ポジションなし       |
| `black`  | 0.0x      | 緊急 — 全ポジションクローズ         |

## アクション (優先度順)

複数の条件が同時に発火した場合、最も優先度の高いアクションが選択されます。

```
優先度  アクション           効果
──────────────────────────────────────────────────────────
  6     stop_bot           全ポジションクローズ、ボット停止
  5     force_close        全ポジションクローズ、緊急停止記録
  4     close_all          全ポジションクローズ (RISK_LEVEL=black)
  3     pause              全注文キャンセル、待機 (RISK_LEVEL=red)
  2     cooldown           クールダウンタイマー満了まで待機
  1     block_new_orders   既存ポジション管理のみ、新規注文なし
  0     none               通常取引
```

## トリガー条件

### メトリクスベースのチェック (毎サイクル評価)

| チェック項目        | 条件                                               | デフォルト閾値 |
|--------------------|----------------------------------------------------|----------------|
| レバレッジ          | `leverage > max_leverage`                          | 3.0            |
| マージン比率        | `margin_ratio >= max_margin_usage`                 | 0.8 (80%)      |
| ドローダウン        | `(開始残高 - 現在) / 開始残高 > max_drawdown`       | 0.1 (10%)      |
| 日次損失 %         | `(当日開始 - 現在) / 当日開始 > limit`              | 0.05 (5%)      |
| 最大ポジション数   | `num_positions > max_open_positions`                | 5              |

### オプトインチェック (デフォルト無効)

| チェック項目         | 条件                                               | 設定キー              |
|---------------------|----------------------------------------------------|----------------------|
| 強制クローズマージン | `margin_ratio >= FORCE_CLOSE_MARGIN`               | `FORCE_CLOSE_MARGIN` |
| 日次損失 (絶対値)   | `daily_loss_usd >= DAILY_LOSS_LIMIT`               | `DAILY_LOSS_LIMIT`   |
| 個別トレードSL      | `trade_loss_pct >= PER_TRADE_STOP_LOSS`            | `PER_TRADE_STOP_LOSS`|

### 安全ガード

1サイクルで残高が50%以上変動した場合、データ異常（例: spot API 429）と判断し、誤発火を防ぐために無視されます。

## 個別トレード ストップロス

メインのリスクチェックとは独立して毎サイクル評価されます:

```
各ポジションについて:
  loss_pct = |未実現PnL| / ポジション価値   (PnL < 0 の場合)
  if loss_pct >= PER_TRADE_STOP_LOSS:
    → そのポジションを成行クローズ
```

## サーキットブレーカー連携

ボットはリスクチェックとストラテジー実行をサーキットブレーカーで保護しています:

- **risk_metrics**: `check_risk_limits()` が5サイクル以上連続で「メトリクスなし」を返した場合、全注文をキャンセル。
- **strategy**: ストラテジー実行が5回以上連続でエラーになった場合、自動復帰（60秒）までシグナル生成をスキップ。

## 復帰パス

| 状態              | 復帰条件                                              |
|-------------------|-------------------------------------------------------|
| block_new_orders  | 全メトリクスチェックが再び合格                        |
| pause             | RISK_LEVEL が `red` から `green`/`yellow` に変更      |
| cooldown          | `cooldown_after_stop` 秒経過 (デフォルト 3600)        |
| close_all         | RISK_LEVEL が `black` から変更                         |
| force_close       | マージン比率が閾値未満 + クールダウン満了              |
| stop_bot          | 手動再起動が必要                                      |

## トレーディングループのフロー

```
_trading_loop():
  1. check_risk_limits()
  2. いずれかのチェックが失敗した場合:
       cancel_all_orders()
       最優先アクションを実行 (stop_bot > force_close > close_all > ...)
       return
  3. update_order_status()
  4. check_per_trade_stops()     ← 個別の損失ポジションをクローズ
  5. strategy.run(coins)          ← シグナル生成 + 注文発注
  6. リスクサマリーをログ出力 (60秒ごと)
```
